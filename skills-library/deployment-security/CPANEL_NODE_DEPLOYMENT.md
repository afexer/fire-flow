# cPanel Node.js Application Deployment via PHP Installer

## Overview
Deploy Node.js applications on shared cPanel hosting using a WordPress-style PHP installer wizard. This approach enables non-technical users to deploy complex Node.js/React applications without command line access.

## Architecture

### Two-Phase Deployment
1. **PHP Installer (Web-Based)** - Collects configuration, tests connections, generates .env
2. **Node.js Server** - Runs via cPanel's Node.js Selector or PM2

### Directory Structure on cPanel
```
public_html/
├── install/                 # PHP installer wizard (delete after install)
│   ├── index.php           # Main entry with AJAX handlers
│   ├── includes/
│   │   ├── requirements.php
│   │   └── functions.php
│   └── steps/
│       └── step1-13.php
├── server/                  # Node.js backend
│   ├── server.js
│   ├── package.json
│   ├── .env                # Generated by installer
│   └── database/
│       └── adapters/       # MySQL/PostgreSQL support
└── client/                  # React frontend (built)
    └── dist/
```

## SSH Deployment Commands

### Initial Upload
```bash
# Upload installer
scp -i "path/to/ssh-key" -r install/ user@host:/home/user/public_html/

# Upload server files
scp -i "path/to/ssh-key" -r server/ user@host:/home/user/public_html/

# Upload built client
scp -i "path/to/ssh-key" -r client/dist/ user@host:/home/user/public_html/client/
```

### Database Setup via cPanel uapi
```bash
# Create database
uapi Mysql create_database name="user_dbname"

# Create user
uapi Mysql create_user name="user_dbuser" password="SecurePass123!"

# Grant privileges
uapi Mysql set_privileges_on_database user="user_dbuser" database="user_dbname" privileges="ALL PRIVILEGES"
```

### Node.js Setup
```bash
# Install dependencies
cd ~/public_html/server && npm install

# Start with PM2
pm2 start server.js --name "your-app-server"
pm2 save
```

### Key PHP Functions
```php
// Test database connection
function testDatabaseConnection($config) {
    $dbType = $config['db_type'] ?? 'mysql';
    $dsn = $dbType === 'mysql'
        ? "mysql:host={$host};port={$port};dbname={$name};charset=utf8mb4"
        : "pgsql:host={$host};port={$port};dbname={$name}";

    $pdo = new PDO($dsn, $user, $pass, [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_TIMEOUT => 5
    ]);
    return ['success' => true];
}

// Generate .env file
function generateEnvFile($config) {
    $content = "DB_TYPE=" . $config['db_type'] . "\n";
    $content .= "DB_HOST=" . $config['db_host'] . "\n";
    // ... more variables
    $content .= "JWT_SECRET=" . bin2hex(random_bytes(32)) . "\n";
    return file_put_contents('../server/.env', $content);
}
```

## Database Abstraction Layer

### Knex.js Cross-Database Support
```javascript
// server/database/config.js
const dbType = process.env.DB_TYPE || 'pg';
const isSupabase = process.env.DATABASE_URL?.includes('supabase');

const config = {
  client: dbType === 'mysql' ? 'mysql2' : 'pg',
  connection: {
    host: process.env.DB_HOST,
    port: process.env.DB_PORT,
    database: process.env.DB_NAME,
    user: process.env.DB_USER,
    password: process.env.DB_PASS,
    ssl: isSupabase ? { rejectUnauthorized: false } : false
  }
};
```

### Adapter Pattern
```javascript
// PostgreSQL: JSONB, ILIKE, upsert with ON CONFLICT
// MySQL: JSON columns, CONCAT_WS, REPLACE INTO
class PostgresAdapter extends BaseAdapter {
  async search(table, column, query) {
    return this.db(table).where(column, 'ilike', `%${query}%`);
  }
}

class MySQLAdapter extends BaseAdapter {
  async search(table, column, query) {
    return this.db(table).where(column, 'like', `%${query}%`);
  }
}
```

## Troubleshooting

### Common Issues
1. **Node.js not found** - Use cPanel Node.js Selector to register app
2. **Database connection refused** - Check if localhost or 127.0.0.1 is required
3. **Permission denied** - Ensure 755 for directories, 644 for files
4. **SSL errors with Supabase** - Add `ssl: { rejectUnauthorized: false }`

### DNS Configuration
- Remove any conflicting nameservers (e.g., Vercel)
- Point A record to cPanel server IP
- Allow 24-48 hours for propagation

## Security Reminders
1. Delete `/install` folder after successful installation
2. Set restrictive permissions on `.env` file (600)
3. Use HTTPS in production
4. Enable cPanel firewall rules
